## 1. 产品概述

### 1.1 产品愿景

打造一个**现代化、智能化、有温度**的AI命理服务平台，将传统命理文化与前沿AI技术深度融合，为用户提供专业、私密、便捷的命理咨询服务。

### 1.2 产品定位

| 维度 | 描述 |
|------|------|
| **目标市场** | 中国大陆及海外华人市场 |
| **产品形态** | 响应式Web应用（移动端优先） |
| **核心价值** | AI驱动的个性化命理分析 + 情感陪伴式对话体验 |
| **差异化优势** | 多人格AI、记忆式对话、多命理体系、隐私优先 |

### 1.3 品牌资产

| 资产 | 说明 |
|------|------|
| **产品名称** | MingAI（已确定）|
| **Logo** | [Logo.png](file:///Users/hhs/Develop/Project/MingAI/Logo.png)（已设计完成）|

### 1.4 目标用户画像

| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| **好奇探索者** | 18-30岁，初次接触命理 | 免费试用、趣味性强 |
| **生活困惑者** | 25-45岁，面临人生抉择 | 专业分析、即时决策建议、情感支持 |
| **命理爱好者** | 30-55岁，有命理基础 | 深度解读、专业术语、连续追问 |
| **商务决策者** | 30-50岁，创业者/管理者 | 合伙评估、时机选择、保密性强 |

## 3. 技术架构

### 3.1 技术栈

| 类别 | 技术选型 | 说明 |
|------|----------|------|
| **框架** | Next.js 16+ (App Router) | 全栈框架，Server Actions处理后端逻辑 |
| **语言** | TypeScript | 前后端统一，严格类型定义 |
| **UI** | Tailwind CSS + Lucide React | 移动端优先，响应式设计 |
| **数据库** | Supabase (PostgreSQL) | ✅ 项目已创建 |
| **八字算法** | lunar-javascript | 成熟的农历/八字计算库 |
| **AI模型** | DeepSeek + GLM-4.6 | ✅ API已准备 |
| **支付** | 微信支付 + 支付宝 | 覆盖主流支付方式 |
| **部署** | Vercel (免费) | 与Next.js无缝集成 |

### 3.2 环境变量配置
已在/docs/一些配置.md中说明。
```env
# AI API 配置（请自行填写）
DEEPSEEK_API_KEY=your_deepseek_api_key_here
GLM_API_KEY=your_glm_api_key_here

# Supabase 配置（请自行填写）
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url_here
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here

# 微信登录配置（请自行填写）
WECHAT_APP_ID=your_wechat_app_id_here
WECHAT_APP_SECRET=your_wechat_app_secret_here
```

### 3.3 AI模型策略

| 场景 | 推荐模型 | 原因 |
|------|----------|------|
| 深度解读 | DeepSeek | 中文理解能力强，长文本推理佳 |
| 日常对话 | GLM-4.6 | 响应速度快，对话流畅 |
| 简短分析 | DeepSeek-Lite | 成本低，适合批量生成 |

### 3.4 部署方案

> [!TIP]
> 推荐使用 **Vercel** 免费部署，与Next.js完美兼容。

| 平台 | 费用 | 优势 | 劣势 |
|------|------|------|------|
| **Vercel** ⭐推荐 | 免费（Hobby） | 与Next.js无缝集成、自动HTTPS、全球CDN | 免费版有带宽限制 |
| Netlify | 免费 | 类似Vercel | Next.js支持略逊 |
| Railway | 免费额度 | 支持后端服务 | 免费额度有限 |
| GitHub Pages | 免费 | 完全免费 | 仅支持静态站点 |

**部署步骤（Vercel）**:
1. 将代码推送到GitHub仓库
2. 访问 [vercel.com](https://vercel.com) 并用GitHub登录
3. 导入项目，自动检测Next.js配置
4. 添加环境变量
5. 点击部署，获取 `your-project.vercel.app` 域名

> 必须支持**深色/浅色主题切换**，用户可自由选择。

| 主题 | 配色方案 |

| **深色模式** | 背景 #0a0a0a / #1a1a1a，文字 #ffffff / #a0a0a0 |
