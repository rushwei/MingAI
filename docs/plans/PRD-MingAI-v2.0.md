## 1. 产品概述

### 1.1 产品愿景

打造一个**现代化、智能化、有温度**的AI命理服务平台，将传统命理文化与前沿AI技术深度融合，为用户提供专业、私密、便捷的命理咨询服务。

### 1.2 产品定位

| 维度           | 描述                                        |
| -------------- | ------------------------------------------- |
| **目标市场**   | 中国大陆及海外华人市场                      |
| **产品形态**   | 响应式Web应用（移动端优先）                 |
| **核心价值**   | AI驱动的个性化命理分析 + 情感陪伴式对话体验 |
| **差异化优势** | 流式对话、记忆式交互、多命理体系、隐私优先  |

### 1.3 品牌资产

| 资产         | 说明                                |
| ------------ | ----------------------------------- |
| **产品名称** | MingAI（已确定）                    |
| **Logo**     | [Logo.png](/Logo.png)（已设计完成） |

### 1.4 目标用户画像

| 用户类型       | 特征                   | 核心需求                         |
| -------------- | ---------------------- | -------------------------------- |
| **好奇探索者** | 18-30岁，初次接触命理  | 免费试用、趣味性强               |
| **生活困惑者** | 25-45岁，面临人生抉择  | 专业分析、即时决策建议、情感支持 |
| **命理爱好者** | 30-55岁，有命理基础    | 深度解读、专业术语、连续追问     |
| **商务决策者** | 30-50岁，创业者/管理者 | 合伙评估、时机选择、保密性强     |

## 3. 技术架构

### 3.1 技术栈

| 类别         | 技术选型                    | 说明          |
| ------------ | --------------------------- | ------------- |
| **框架**     | Next.js 16+ (App Router)    | 全栈框架      |
| **语言**     | TypeScript                  | 严格类型定义  |
| **UI**       | Tailwind CSS + Lucide React | 响应式设计    |
| **数据库**   | Supabase (PostgreSQL)       | ✅ 已配置      |
| **八字算法** | lunar-javascript            | 农历/八字计算 |
| **紫微算法** | iztro                       | 紫微斗数计算  |
| **AI模型**   | DeepSeek/GLM/Gemini/Qwen/专属分析模型DeepAI          | ✅ 流式输出    |
| **部署**     | Vercel + Zeabur                  | 自动部署      |
