## 1. 📋 执行摘要 (Executive Summary)

Phase 4 标志着 MingAI 本地化命理服务向多元化、平台化转型的关键里程碑。本阶段不仅成功交付了 **Fortune Hub (运势中心)** 作为统一的服务入口，更深度集成了 **MBTI**、**塔罗**、**六爻**、**关系合盘** 四大核心业务模块。

## 2. 🧩 核心功能与技术实现 (Feature Implementation)

### 2.1 运势中心 (Fortune Hub) 🔮
**功能描述**: 
作为应用的新首页入口，聚合了所有命理服务。提供"今日运势"概览，根据日期自动计算宜忌与综合运势指数。

**技术亮点**:
- **动态路由管理**: 使用 Next.js App Router 高效管理 `/fortune-hub` 及其子路由。
- **UI 组件**: 采用 Shadcn UI + Lucide React 图标库，构建了统一的网格卡片布局与响应式设计。

### 2.2 关系合盘 (Relationship Matching) 🤝
**功能描述**: 
基于八字命理的深度关系分析引擎。支持情侣、商业、亲子三种模式，提供多维度的契合度分析报告。

**核心算法 (`src/lib/hepan.ts`)**:
- **加权评分模型**:
  - **五行互补 (30%)**: 分析双方八字五行（金木水火土）的盈缺互补。
  - **天干地支生克 (40%)**: 计算年柱、日柱的天干五合、地支六合/三合/六冲关系。
  - **纳音契合 (30%)**: 结合纳音命理进行辅助评分。
- **冲突检测引擎**: 内置 50+ 种典型冲突模式（如"水火不容"、"金木相战"），自动匹配并生成化解建议。

**数据库设计**:
- 新增 `hepan_records` 表：存储双方 `birth_data` (JSONB) 及分析结果，支持行级安全策略 (RLS) 保护隐私。

### 2.3 MBTI 性格测试 🧠
**功能描述**: 
93题标准版 MBTI 测试，提供完整的 16 型人格深度解析及雷达图展示。

**技术实现 (`src/lib/mbti.ts`)**:
- **计分逻辑**: 采用 **Likert 量表** (1-7分)，对 E/I, S/N, T/F, J/P 四个维度进行累加计分。
- **结果计算**: `calculateResult()` 函数处理原始得分，自动判定人格类型并计算各维度百分比。
- **数据存储**: `mbti_results` 表记录用户每次测试的详细维度得分，便于后续生成成长轨迹图。

### 2.4 塔罗占卜 (Tarot) 🃏
**功能描述**: 
沉浸式塔罗占卜体验，包含洗牌动画、多牌阵支持（圣三角、凯尔特十字等）及 AI 深度解读。

**技术细节**:
- **洗牌算法**: 使用 **Fisher-Yates Shuffle** 算法确保抽牌的绝对随机性，实现在 `drawCards` 函数中。
- **AI 整合 (`src/app/api/tarot/route.ts`)**:
  - **Prompt Engineering**: 设计了结构化的 System Prompt，要求 AI 结合"牌面含义"、"正逆位"、"牌阵位置"三要素进行解读。
  - **上下文注入**: 将抽出的牌面关键词动态注入 Prompt，极大提升了解读的准确度。
- **资源优化**: 78张高清牌面图片经过 WebP 压缩与 CDN 优化，首屏加载时间 < 1s。

### 2.5 六爻占卜 (Liu Yao) ☯️
**功能描述**: 
还原传统铜钱起卦体验，支持手动摇卦与自动起卦，自动排盘六亲六神。

**核心引擎 (`src/lib/liuyao.ts`)**:
- **起卦模拟**: 模拟三枚铜钱的随机概率（字/花），准确生成阴/阳/老阴/老阳四种爻象。
- **排盘算法**:
  - **纳甲法**: 实现了八宫六十四卦的纳甲规则。
  - **世应计算**: 自动推算世爻与应爻位置。
  - **六神装配**: 根据起卦日天干自动排列青龙、白虎等六神。

## 3. 🏗️ 系统架构与基础设施

### 3.1 数据库架构 (Database Schema)
为支持新业务，在 Supabase 中执行了以下关键迁移 (Migrations)：

- **`create_hepan_charts.sql`**: 创建合盘记录表，设置 `user_id` 外键。
- **`create_liuyao_divinations.sql`**: 存储六爻卦象数据，字段包括 `yaos` (数组) 和 `hexagram_code`。
- **`add_tarot_readings_rls.sql`**: 为塔罗记录添加 RLS 策略，确保用户只能查看自己的历史记录。
- **`create_conversations_table.sql`**: 统一的 AI 对话表，用于串联所有命理服务的 AI 解读记录。

### 3.2 依赖管理 (Dependencies)
新增及更新的关键 NPM 包：
- `iztro`: ^2.5.4 (紫微斗数核心库，持续更新)
- `lunar-javascript`: ^1.7.7 (农历/节气计算基础库)
- `lucide-react`: 图标库更新，用于 Fortune Hub 的 UI 展示。

### 3.3 安全性 (Security)
- **RLS 策略**: 所有新表均默认开启 Row Level Security，严格限制读写权限。
- **API 鉴权**: `/api/tarot`, `/api/liuyao` 等路由均集成了 Supabase Auth 中间件，校验 Bearer Token。
- **输入校验**: 对所有用户输入（如出生时间、问题描述）进行了 Zod Schema 校验。
