# 首页默认页与用户中心展示修复

## 1. 背景与问题

- 当前现状：
  - 根路由 `/` 固定重定向到 `/fortune-hub`。
  - Linux.do 登录后，用户中心直接显示 `auth.users.email`，当上游返回隐私中继邮箱时，页面会显示一串难以识别的 relay 地址。
  - 侧边栏与移动端导航首屏采用客户端 fail-open 默认值，刷新时会短暂显示被管理员关闭或隐藏的入口，随后再消失。
- 触发原因：
  - 产品希望将已登录用户的默认落点调整为用户中心。
  - Linux.do 上游邮箱可能是 `privaterelay.linux.do` 形式的隐私地址，不适合直接作为用户可读展示。
  - 功能开关与侧边栏配置都依赖客户端异步拉取，组件初始渲染未等待加载完成。
- 主要痛点：
  - 首页落点不符合当前产品主流程。
  - 用户中心邮箱显示体验差，容易让用户误判成系统数据错误。
  - 导航入口闪烁暴露了本应隐藏的功能，影响观感与可信度。

## 2. 目标与非目标

### 2.1 目标

- 已登录用户访问 `/` 时默认进入 `/user`，未登录用户仍保留 `/fortune-hub`。
- Linux.do relay 邮箱在用户中心与资料页中改为友好展示，不再直接暴露 relay 字符串。
- 刷新时桌面侧边栏与移动端导航不再先显示被关闭/隐藏的入口。

### 2.2 非目标

- 不调整 Linux.do OAuth 协议与建号流程。
- 不修改数据库结构。
- 不重做用户中心视觉样式。

## 3. 方案设计

### 3.1 变更范围

- 影响模块：
  - `src/app/page.tsx`
  - `src/app/user/page.tsx`
  - `src/app/user/profile/page.tsx`
  - `src/components/profile/EmailSection.tsx`
  - `src/components/layout/Sidebar.tsx`
  - `src/components/layout/MobileNav.tsx`
  - `src/components/layout/UserMenu.tsx`
  - `src/lib/*` 中新增用户邮箱展示辅助逻辑
  - `src/tests/*` 中新增/更新回归测试
- 影响接口：
  - 不新增外部 API。
  - 根路由的默认跳转行为发生变化。
- 影响数据结构：
  - 无。

### 3.2 实现方案

- 核心思路：
  - 首页根据当前服务端会话决定默认跳转目标。
  - 将“用户可读邮箱展示”与“真实认证邮箱值”分离；对 Linux.do relay 邮箱使用统一友好文案。
  - 导航组件在功能开关与侧边栏配置完成加载前，先渲染稳定占位而不是 fail-open 全量入口。
- 关键流程：
  - `/` 读取当前会话，已登录跳 `/user`，否则跳 `/fortune-hub`。
  - 用户中心、个人资料页、用户菜单统一通过邮箱展示 helper 渲染文案。
  - `Sidebar` / `MobileNav` 联合使用 `useFeatureToggles().isLoading` 与 `useSidebarConfigSafe().loading` 控制首屏占位。
- 兼容策略（向前/向后兼容）：
  - 不改变真实 `auth.users.email` 与邮箱修改流程。
  - 仅改变展示层文案与导航加载时机。

### 3.3 数据库与迁移（如适用）

- 是否需要 migration：否。
- 涉及对象（表/字段/索引/约束）：无。
- 回填策略：无。
- RLS/权限影响：无。

## 4. 验收标准

- 功能验收：
  - 已登录用户访问 `/` 进入 `/user`。
  - 未登录用户访问 `/` 仍进入 `/fortune-hub`。
  - Linux.do relay 邮箱不再在用户中心/资料页直接显示原始 relay 字符串。
  - 刷新页面时，管理员关闭/隐藏的导航入口不会先出现后消失。
- 接口验收：
  - 无新增 API 契约变化。
- 性能/稳定性验收：
  - 导航加载期间使用稳定占位，不因短暂 loading 造成布局跳闪失控。
- 回归范围：
  - 首页跳转逻辑。
  - 用户邮箱展示 helper。
  - 导航加载占位逻辑。

## 5. 测试计划

- 单元测试：
  - 邮箱展示 helper 对 relay 邮箱与普通邮箱的输出。
- 集成/路由测试：
  - 根路由跳转目标。
  - 侧边栏/移动端导航在 loading 态下不直接渲染入口。
- 手动验证步骤：
  - 登录状态访问 `/`，确认跳转到 `/user`。
  - Linux.do 登录后查看用户中心与资料页邮箱展示。
  - 刷新桌面与移动端页面，确认隐藏入口不闪现。

## 6. 风险与回滚

- 失败信号：
  - 首页跳转循环。
  - 普通邮箱被误判成 relay 文案。
  - 导航 loading 占位长期不消失。
- 风险等级：中。
- 回滚步骤：
  - 回退根路由重定向改动。
  - 回退邮箱展示 helper 接入。
  - 回退导航 loading 占位逻辑。

## 7. 里程碑与任务拆分

- M1：补失败测试，锁定目标行为。
- M2：首页跳转与邮箱展示修复。
- M3：导航 loading 占位落地并回归验证。
