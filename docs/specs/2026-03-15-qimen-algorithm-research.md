## 一、基础概念

### 1.1 洛书九宫结构

奇门遁甲基于洛书九宫格，将空间分为九个宫位，对应八卦方位：

```
┌─────────┬─────────┬─────────┐
│ 4-巽宫   │ 9-离宫   │ 2-坤宫   │
│ 东南/木  │ 正南/火  │ 西南/土  │
├─────────┼─────────┼─────────┤
│ 3-震宫   │ 5-中宫   │ 7-兑宫   │
│ 正东/木  │ 中央/土  │ 正西/金  │
├─────────┼─────────┼─────────┤
│ 8-艮宫   │ 1-坎宫   │ 6-乾宫   │
│ 东北/土  │ 正北/水  │ 西北/金  │
└─────────┴─────────┴─────────┘
```

九宫编号与属性：

| 9 | 离 | 正南 | 火 |

### 1.2 四层盘面结构

奇门遁甲盘面由四层叠加组成：

1. **地盘**（Earth Plate）：固定不动，根据局数排布三奇六仪
2. **天盘**（Heaven Plate）：九星随值符转动
3. **人盘**（Human Plate / 八门）：八门随值使转动
4. **神盘**（Deity Plate / 八神）：八神跟随值符宫位排布

### 1.3 三奇六仪

十天干中，甲为统帅，隐遁不现，藏于六仪之下：

- **三奇**：乙（日奇）、丙（月奇）、丁（星奇）
- **六仪**：戊、己、庚、辛、壬、癸

甲隐遁对应关系（六甲旬首）：

| 旬首 | 甲隐于 | 六仪 |

| 甲子 | 戊 | 甲子戊 |
| 甲戌 | 己 | 甲戌己 |
| 甲申 | 庚 | 甲申庚 |
| 甲午 | 辛 | 甲午辛 |
| 甲辰 | 壬 | 甲辰壬 |
| 甲寅 | 癸 | 甲寅癸 |

地盘排布的固定顺序（九个干）：**戊、己、庚、辛、壬、癸、丁、丙、乙**

### 1.4 洛书飞布顺序

八个外宫的顺时针遍历顺序（跳过中宫5）：

```
顺时针：1 → 8 → 3 → 4 → 9 → 2 → 7 → 6
逆时针：1 → 6 → 7 → 2 → 9 → 4 → 3 → 8
```

## 二、定局算法（拆补法）

### 2.1 阳遁与阴遁

- **阳遁**：冬至后到夏至前（约12月22日 ~ 6月21日），局数顺排 1→9
- **阴遁**：夏至后到冬至前（约6月21日 ~ 12月22日），局数逆排 9→1

### 2.2 节气与局数对应表（核心数据表）

每个节气对应上元、中元、下元三个局数。编码时用3位字符串表示，如 "174" 表示上元=1局，中元=7局，下元=4局。

### 2.3 上中下元的确定方法（拆补法）

拆补法通过日干支的地支来确定当前属于上元、中元还是下元：

**规则：查看当日日干支的地支（即日柱地支）：**

| 日柱地支 | 所属元 |
|----------|--------|
| 子、午、卯、酉 | 上元 |
| 寅、申、巳、亥 | 中元 |
| 辰、戌、丑、未 | 下元 |

**编码实现：**
```
地支 = getDayEarthlyBranch(date)
if 地支 in [子,午,卯,酉]: yuanIndex = 0  // 上元
if 地支 in [寅,申,巳,亥]: yuanIndex = 1  // 中元
if 地支 in [辰,戌,丑,未]: yuanIndex = 2  // 下元
```

### 2.4 定局完整算法流程

```
输入：公历日期时间 (Date)
输出：局数 (1-9)、阴阳遁类型

1. 将公历转换为农历/干支历
2. 获取当前所处节气名称（取最近的前一个节气）
3. 判断阴阳遁：
   - 冬至后 ~ 夏至前 → 阳遁
   - 夏至后 ~ 冬至前 → 阴遁
4. 获取日柱地支，确定上中下元 (yuanIndex = 0/1/2)
5. 查节气局数表，取对应编码的第 yuanIndex 位数字
6. 该数字即为局数
```

### 2.5 拆补法 vs 置润法

| 特性 | 拆补法 | 置润法 |
|------|--------|--------|
| 上中下元判断 | 根据日柱地支 | 根据符头（旬首甲日） |
| 节气边界处理 | 跨节气时"拆"旧节气、"补"新节气 | 设置闰局处理 |
| 使用广泛度 | 最主流，推荐使用 | 较少使用 |
| 实现复杂度 | 较简单 | 需要额外闰局逻辑 |

**推荐实现拆补法**，这是当前最主流的定局方法。

## 三、排盘步骤（转盘法）

### 3.1 步骤一：排地盘

地盘是固定的，根据局数确定九个天干在九宫中的位置。

**固定干序**：戊、己、庚、辛、壬、癸、丁、丙、乙（共9个，甲隐遁不上盘）

**排布规则**：
- 阳遁：从局数对应的宫位开始，按洛书顺序（1→8→3→4→9→2→7→6）顺排
- 阴遁：从局数对应的宫位开始，按洛书逆序（1→6→7→2→9→4→3→8）逆排

**算法伪代码**：
```
stemOrder = [戊, 己, 庚, 辛, 壬, 癸, 丁, 丙, 乙]
luoshuCW  = [1, 8, 3, 4, 9, 2, 7, 6]  // 顺时针（不含中宫5）
luoshuCCW = [1, 6, 7, 2, 9, 4, 3, 8]  // 逆时针

function getDiPan(type, juNumber):
    result = {}  // palace -> stem
    sequence = (type == '阳遁') ? luoshuCW : luoshuCCW

    // 找到局数对应宫位在序列中的起始索引
    startIndex = sequence.indexOf(juNumber)
    // 如果局数是5（中宫），寄坤二宫
    if juNumber == 5: startIndex = sequence.indexOf(2)

    for i in 0..8:
        if i < 8:
            palace = sequence[(startIndex + i) % 8]
        else:
            palace = 5  // 第9个干放中宫
        result[palace] = stemOrder[i]

    return result
```

**示例：阳遁一局**
- 戊从1宫开始，按顺序：戊→1宫, 己→8宫, 庚→3宫, 辛→4宫, 壬→9宫, 癸→2宫, 丁→7宫, 丙→6宫, 乙→中5宫（实际寄2宫）

### 3.2 步骤二：确定旬首与值符值使

**确定时柱旬首**：

六十甲子分为六旬，每旬以甲开头：
```
甲子旬：甲子 乙丑 丙寅 丁卯 戊辰 己巳 庚午 辛未 壬申 癸酉
甲戌旬：甲戌 乙亥 丙子 丁丑 戊寅 己卯 庚辰 辛巳 壬午 癸未
甲申旬：甲申 乙酉 丙戌 丁亥 戊子 己丑 庚寅 辛卯 壬辰 癸巳
甲午旬：甲午 乙未 丙申 丁酉 戊戌 己亥 庚子 辛丑 壬寅 癸卯
甲辰旬：甲辰 乙巳 丙午 丁未 戊申 己酉 庚戌 辛亥 壬子 癸丑
甲寅旬：甲寅 乙卯 丙辰 丁巳 戊午 己未 庚申 辛酉 壬戌 癸亥
```

**旬首对应六仪**：
```
xunShouMap = {
  甲子: 戊, 甲戌: 己, 甲申: 庚,
  甲午: 辛, 甲辰: 壬, 甲寅: 癸
}
```

**确定值符（Duty Star）**：
1. 获取时柱干支，确定其所属旬首
2. 查旬首对应的六仪（如甲子旬→戊）
3. 在地盘上找到该六仪所在的宫位 → 该宫位的原始九星即为**值符**
4. 如果六仪落在中宫5，则寄坤二宫

**确定值使（Duty Gate）**：
- 值符所在宫位的原始八门即为**值使**

### 3.3 步骤三：排天盘（九星转动）

天盘采用**转盘法**：所有九星作为整体旋转，保持相对位置不变。

**算法**：
1. 找到值符星的原始宫位（值符宫 zhiFuGong）
2. 找到时干在地盘上的落宫（时干落宫 shiGanGong）
   - 如果时干是甲，甲隐遁不现，则时干落宫 = 值符宫
3. 计算旋转步数：
   ```
   zhiFuIndex = luoshuCW.indexOf(zhiFuGong)
   shiGanIndex = luoshuCW.indexOf(shiGanGong)
   steps = (shiGanIndex - zhiFuIndex + 8) % 8
   ```
4. 所有九星按相同步数旋转：
   ```
   for i in 0..7:
       originalPalace = luoshuCW[i]
       newPalace = luoshuCW[(i + steps) % 8]
       tianPan[newPalace] = 原始星[originalPalace]
   ```
5. **天禽星特殊处理**：天禽原在中宫5，不参与旋转，跟随天芮（2宫星）。天芮到哪个宫，天禽就寄在哪个宫，显示为"禽芮"。中宫5始终为空。

### 3.4 步骤四：排八门（值使转动）

八门的转动也采用转盘法，但有一个关键区别：**门的转动步数是星的2倍**。

**算法**：
1. 确定值使门（值符宫的原始门）
2. 找到时干在地盘上的落宫
3. 计算旋转步数：
   ```
   baseSteps = (shiGanIndex - zhiFuIndex + 8) % 8
   gateSteps = (baseSteps * 2) % 8
   ```
4. 阳遁顺转，阴遁逆转：
   ```
   // 阳遁
   for i in 0..7:
       newIndex = (i + gateSteps) % 8
       newPalace = luoshuCW[newIndex]
       baMen[newPalace] = 原始门[luoshuCW[i]]

   // 阴遁
   for i in 0..7:
       newIndex = (i - gateSteps + 8) % 8
       newPalace = luoshuCW[newIndex]
       baMen[newPalace] = 原始门[luoshuCW[i]]
   ```
5. 中宫5没有门。

> **注意**：关于八门转动步数是否为九星的2倍，不同流派有不同说法。上述是 qfdk/qimen 实现的方式。另一种常见实现是八门与九星使用相同步数，只是值使门从值使宫转到时干落宫。建议实现时提供配置选项。

### 3.5 步骤五：排八神

八神从值符落宫开始，按固定顺序排布。

**八神顺序**：值符、腾蛇、太阴、六合、白虎（勾陈）、玄武（朱雀）、九地、九天

**排布规则**：
- 阳遁：从值符落宫开始，按洛书**顺时针**排布
- 阴遁：从值符落宫开始，按洛书**逆时针**排布

**算法**：
```
shenOrder = [值符, 腾蛇, 太阴, 六合, 白虎, 玄武, 九地, 九天]

function distributeBaShen(zhiFuLuoGong, type):
    result = {}
    sequence = (type == '阳遁') ? luoshuCW : luoshuCCW

    // 值符落宫如果是5（中宫），寄坤二宫
    startGong = (zhiFuLuoGong == 5) ? 2 : zhiFuLuoGong
    startIndex = sequence.indexOf(startGong)

    for i in 0..7:
        palace = sequence[(startIndex + i) % 8]
        result[palace] = shenOrder[i]

    return result
```

### 3.6 步骤六：排暗干

## 七、格局判断

### 7.1 基础格局

**伏吟（Fu Yin）**：
- 星伏吟：天盘九星回到自己的原始宫位（如天蓬在1宫、天芮在2宫...）
- 门伏吟：八门回到自己的原始宫位
- 判断条件：`天盘星[宫] == 该宫原始星` 或 `八门[宫] == 该宫原始门`
- 含义：事情停滞不前，主守不主攻

**反吟（Fan Yin）**：
- 星反吟：天盘九星到了与原始宫位对冲的宫位（如天蓬到9宫、天芮到8宫...）
- 门反吟：八门到了与原始宫位对冲的宫位
- 对冲关系：1↔9, 2↔8, 3↔7, 4↔6
- 含义：事情反复无常

**空亡（Kong Wang）**：
- 六十甲子每旬10个干支，12地支中有2个地支空出，即为空亡
- 旬空对应表：
  ```
  甲子旬：戌亥空
  甲戌旬：申酉空
  甲申旬：午未空
  甲午旬：辰巳空
  甲辰旬：寅卯空
  甲寅旬：子丑空
  ```
- 地支对应宫位：子→1, 丑→8, 寅→8, 卯→3, 辰→4, 巳→4, 午→9, 未→2, 申→2, 酉→7, 戌→6, 亥→6
- 日空亡：根据日柱旬首确定
- 时空亡：根据时柱旬首确定

**入墓（Ru Mu）**：
- 天干入墓到特定宫位
- 入墓对应：甲乙→未(2宫), 丙丁→戌(6宫), 庚辛→丑(8宫), 壬癸→辰(4宫), 戊→戌(6宫)
- 判断：天盘干或地盘干落入其墓库宫位

**击刑（Ji Xing）**：
- 门所在宫位与门的原始宫位构成地支相刑关系
- 三刑关系：寅→巳→申→寅, 丑→戌→未→丑, 子→卯→子, 辰辰自刑 等

### 7.2 奇仪格局（天干组合）

以下格局通过天盘干与地盘干的组合来判断：

**吉格**：

```
score = 0

if 门.吉凶 == '凶': score -= 1
```

## 九、驿马与空亡

### 9.1 驿马计算

驿马根据日柱地支的三合局来确定：

| 日支三合 | 驿马 | 对应宫位 |
|----------|------|----------|
| 寅午戌 | 申 | 2宫（坤） |
| 申子辰 | 寅 | 8宫（艮） |
| 巳酉丑 | 亥 | 6宫（乾） |
| 亥卯未 | 巳 | 4宫（巽） |

**算法**：
```
function getYiMa(dayBranch):
    if dayBranch in [寅,午,戌]: return {branch: 申, palace: 2}
    if dayBranch in [申,子,辰]: return {branch: 寅, palace: 8}
    if dayBranch in [巳,酉,丑]: return {branch: 亥, palace: 6}
    if dayBranch in [亥,卯,未]: return {branch: 巳, palace: 4}
```

### 9.2 空亡计算

**日空亡**：根据日柱所属旬首确定空亡地支，再映射到宫位。

**时空亡**：根据时柱所属旬首确定。

## 十一、完整排盘流程（编码参考）

```
输入：公历日期时间 (year, month, day, hour, minute)
输出：完整奇门遁甲盘面

Step 1: 日期转换
  - 公历 → 农历/干支历
  - 获取四柱（年柱、月柱、日柱、时柱）的天干地支
  - 获取当前节气

Step 2: 定局
  - 判断阴阳遁（冬至后阳遁，夏至后阴遁）
  - 根据日柱地支确定上中下元
  - 查节气局数表获取局数 (1-9)

Step 3: 排地盘
  - 按局数和阴阳遁，将 [戊己庚辛壬癸丁丙乙] 排入九宫

Step 4: 确定旬首
  - 根据时柱干支确定所属旬首
  - 查旬首六仪映射表获取对应六仪

Step 5: 确定值符值使
  - 在地盘找到旬首六仪所在宫 → 值符宫
  - 值符宫的原始星 → 值符星
  - 值符宫的原始门 → 值使门

Step 6: 排天盘（九星）
  - 找到时干在地盘的落宫
  - 计算旋转步数
  - 所有九星整体旋转

Step 7: 排八门
  - 值使门转到时干落宫
  - 其余门保持相对位置旋转

Step 8: 排八神
  - 从值符落宫开始
  - 阳遁顺时针、阴遁逆时针排布

Step 9: 计算辅助信息
  - 空亡（日空、时空）
  - 驿马
  - 暗干

Step 10: 格局判断
  - 伏吟/反吟
  - 奇仪格局（天盘干+地盘干组合）
  - 入墓/击刑
  - 宫位综合评分
```

```typescript
interface QimenChart {
```

### 13.2 关键注意事项

1. **中宫寄坤**：中宫5不排星、门、神，天禽寄坤二宫。当值符或时干落中宫时，均寄坤二宫处理。
2. **甲隐遁**：甲不上盘，当时干为甲时，使用其旬首对应的六仪代替。
3. **节气精度**：节气时刻的精确计算直接影响定局准确性，建议使用天文算法（如VSOP87D）而非简单的固定日期。
4. **时辰边界**：中国传统时辰以23:00为子时起点，注意日期切换（23:00后算次日）。
5. **转盘 vs 飞盘**：本文档描述的是转盘法（最主流），飞盘法的星门排布规则不同。
