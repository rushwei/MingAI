# 大六壬排盘功能（Web + MCP）

## 1. 背景与问题

- 当前现状：MingAI 已有八字、紫微斗数、六爻、塔罗等命理功能，但缺少大六壬。
- 触发原因：大六壬与奇门遁甲、太乙神数并称"三式"，是中国古代最重要的占卜术之一，用户需求强烈。
- 主要痛点：市面上大六壬排盘工具多为原生 App，Web 端和 MCP 工具缺乏。

## 2. 目标与非目标

### 2.1 目标

- 实现完整的大六壬排盘核心算法（天地盘、四课、三传、天将盘、遁干、六亲、空亡）
- 提供 Web 排盘页面（起课输入 + 结果展示）
- 提供 MCP 工具（纯计算，无 DB 依赖）
- 集成 AI 解读（流式 SSE）
- 支持历史记录保存与查看

### 2.2 非目标

- 不从零实现排盘算法（使用 `liuren-ts-lib` v1.9.0, Apache 2.0 许可）
- 不实现分类占断的自动生成（由 AI 解读覆盖）
- 不实现金口诀（库支持但首期不做）

#### 天盘核心公式（移植自 Go tianpan.go）

```typescript
// 月将加占时 → 天盘旋转
function up(yueJiang: number, shiZhi: number, diZhi: number): number {
  return (yueJiang + diZhi - shiZhi + 12) % 12;
}
function down(yueJiang: number, shiZhi: number, tianZhi: number): number {
  return (shiZhi + tianZhi - yueJiang + 12) % 12;
}
```

#### 四课取法（移植自 Go sike.go）

```
第一课(干阳): ganYang = up(寄宫(日干))
第二课(干阴): ganYing = up(ganYang)
第三课(支阳): zhiYang = up(日支)
第四课(支阴): zhiYing = up(zhiYang)
```

#### 三传取法优先级（移植自 Go sanchuan.go）

```
1. 伏吟判断: zhiYang == zhi → 取刑为初传
2. 返吟判断: zhiYang 与 zhi 六冲 → 有贼克按贼克取，无则取驿马
3. 贼克法: 下贼上优先于上克下，唯一则取，多个进入比用
4. 八专判断: ganYang == zhiYang → 阳日初传=ganYang+2，阴日初传=zhiYing-2
5. 遥克法: 四课上神与日干有克关系
6. 昴星法: 四课上神全不同且无克 → 阳日取酉上神，阴日取酉下神
7. 别责法: 四课上神有重复但无克 → 阳日取合干寄宫上神，阴日取支+4上神
```

#### 天将盘（移植自 Go tianjiang.go）

```
十二天将: 贵人→螣蛇→朱雀→六合→勾陈→青龙→天空→白虎→太常→玄武→太阴→天后

贵人起法（甲戊庚牛羊）:
  昼贵: [未,申,酉,亥,丑,子,丑,寅,卯,巳] (甲~癸)
  夜贵: [丑,子,亥,酉,未,申,未,午,巳,卯] (甲~癸)

顺逆判断: 贵人所临地盘在巳~戌之间 → 逆布，否则顺布
```

#### 月将确定（节气查表法）

```
中气换将表:
  雨水(330°)→亥(登明)  春分(0°)→戌(河魁)   谷雨(30°)→酉(从魁)
  小满(60°)→申(传送)   夏至(90°)→未(小吉)   大暑(120°)→午(胜光)
  处暑(150°)→巳(太乙)  秋分(180°)→辰(天罡)  霜降(210°)→卯(太冲)
  小雪(240°)→寅(功曹)  冬至(270°)→丑(大吉)   大寒(300°)→子(神后)

使用 lunar-javascript 库获取节气日期，根据占课日期确定当前月将。
```

#### 输入参数设计

```typescript
interface DaliurenInput {
  // 必填
  date: string;           // 公历日期 "YYYY-MM-DD"
  hour: number;           // 时辰 0-23

  // 可选
  question?: string;      // 占事
  yueJiang?: number;      // 月将（0-11，不传则自动计算）
  guiRenMethod?: 'jiawugeng_niuyang' | 'jiayang_wugeng_niu'; // 贵人起法，默认甲戊庚牛羊
  sheHaiMethod?: 'mengzhong' | 'depth';  // 涉害取法，默认孟仲法
  yueJiangMethod?: 'zhongqi' | 'richuan'; // 月将取法，默认中气换将
  diurnal?: boolean;      // 昼夜（不传则根据时辰自动判断）
  birthYear?: number;     // 出生年（用于计算本命和行年）
  gender?: 'male' | 'female'; // 性别（用于计算行年）
}
```

### 3.5 API 路由设计

`POST /api/daliuren`，action 分发：

- `action: 'calculate'` — 纯排盘计算（不消耗积分）
- `action: 'interpret'` — AI 解读（消耗积分，流式 SSE）
- `action: 'save'` — 保存排盘结果
- `action: 'history'` — 查询历史记录

```sql
CREATE TABLE public.daliuren_divinations (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES auth.users(id),
  question text,
  solar_date text NOT NULL,
  lunar_date text,
  day_ganzhi text NOT NULL,
  hour_ganzhi text NOT NULL,

  conversation_id uuid REFERENCES public.conversations(id),
  created_at timestamptz DEFAULT now(),
  PRIMARY KEY (id)
);

-- RLS
ALTER TABLE public.daliuren_divinations ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can view own divinations"
  ON public.daliuren_divinations FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users can insert own divinations"
  ON public.daliuren_divinations FOR INSERT WITH CHECK (auth.uid() = user_id);

-- 索引
CREATE INDEX idx_daliuren_divinations_user_id ON public.daliuren_divinations(user_id);
CREATE INDEX idx_daliuren_divinations_created_at ON public.daliuren_divinations(created_at DESC);
```
